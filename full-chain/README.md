# full-chain

这个项目承载仓库的主线教学页：

`源码规则 -> 构建处理 -> 产物交付 -> 浏览器解析 -> 首屏渲染 -> hydration -> 本地交互更新 -> 请求往返更新`

## 这个项目讲什么

首页按真实发生顺序展示同一个页面的完整旅程：

1. 源码到底在描述什么
2. 构建工具如何处理这份源码
3. 会产出哪些资源
4. 浏览器实际收到的是哪些字节
5. 浏览器如何从 DOM / CSSOM 走到首屏像素
6. hydration 如何让页面从“可见”进入“可交互”
7. 本地点击如何触发状态更新和局部 DOM 更新
8. 什么情况下会走 `fetch -> Route Handler -> JSON -> UI 更新`

## 如何运行

```bash
pnpm install
pnpm dev
```

打开 [http://localhost:3000](http://localhost:3000)。

## 观察点

- 页面顶部先给整条链的总览。
- 中间每段都明确写出输入、输出和职责。
- 刷新能看到服务端首屏时间变化。
- 本地点击只改浏览器内状态。
- 请求按钮会触发一次真实的服务端往返。
